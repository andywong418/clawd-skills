# 2026-03-10 — Memory & Context Recovery

## Context Loss Investigation

### What happened:
- SDK compacted conversation due to context length
- Lost track of what we were discussing
- No session state was preserved before compaction

### Root causes:
1. **No SESSION-STATE.md file** - The WAL protocol says to write critical context here, but file doesn't exist
2. **No working-buffer.md** - Supposed to maintain context at 60% capacity for recovery
3. **Memory files gap** - Last daily memory was March 3rd, 7 days ago
4. **Supermemory underused** - Not being updated frequently with conversation context

## Fixes Implemented

### 1. Max Turns Bug Fixed
- **Bug**: `BOT_MAX_TURNS=200` was being read as 25 because env loaded AFTER module import
- **Fix**: Moved dotenv loading BEFORE other imports in `src/index.ts`
- **Fix**: Changed `agent.ts` to use `getConfig()` function that reads env at runtime
- **Result**: Bot now correctly uses 200 max turns
